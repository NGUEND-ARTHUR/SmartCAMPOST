📦 SMARTCAMPOST – Intelligent Postal Tracking, Pickup & Delivery System (Full Web Application UI)
Author: Nguend Arthur Johann – ICT University, YaoundéProject: SmartCAMPOST – QR-Based Tracking + Geolocation + AI Logistics PlatformGoal: Generate the complete UI/UX for all SmartCAMPOST applications using a clean, modern, enterprise-grade web interface, ready to be implemented using React + TypeScript + TailwindCSS.
🔷 1. Global Concept of the System
SmartCAMPOST is a national-scale intelligent logistics system used by CAMPOST.It enables end-to-end parcel traceability using:
Unique QR Codes
Scans at every operational step
A full timeline of events
OTP verification at pickup/delivery
Geolocation for agents & couriers
AI modules for ETA prediction, route optimization, anomaly detection
Digital payments
The platform includes four different user interfaces:
Client Portal
Agency/Counter Agent Interface
Courier/Delivery Interface
Admin & Supervisor Dashboard
Generate UI screens for each user type, unified by a clean and consistent design system.
🔷 2. Actors & Behaviors to Represent
1. CLIENT (Sender / Receiver)
Uses web portal to:
Create shipment or pickup requests
Describe parcel (content, weight, fragility, declared value)
Choose delivery mode (agency pickup or home delivery)
Track real-time parcel status & timeline
Pay service fees (credit card, Mobile Money, COD)
Receive notifications (SMS/email)
Verify identity via OTP
View receipts, invoices, support tickets
2. AGENCY / COUNTER AGENT
Uses desktop or tablet:
Register parcels dropped in agency
Validate description and weight
Calculate tariff
Generate & print QR label
Accept payment
Scan parcels for:
Acceptance
Arrival / departure in agency
Pickup release (OTP)
3. PICKUP AGENT (Home Collection)
Uses mobile UI:
View scheduled pickup routes
Navigate to client location
Scan temporary or final QR code
Validate parcel information
Confirm collection with GPS position
4. SORTING / HUB AGENT
Uses desktop or rugged mobile device:
Scan parcels at:
Entry origin center
Exit origin
Arrival hub
Departure hub
Arrival destination agency
Manage “bags” or bulk scans
Handle incidents and transit delays
5. COURIER / DELIVERY AGENT
Uses mobile UI:
See delivery route list + optimized route map
Scan “OUT_FOR_DELIVERY”
At delivery:
Trigger OTP to client
Enter OTP code
Upload signature/photo
Mark “DELIVERED” or “FAILED DELIVERY”
6. ADMIN / SUPERVISOR
Uses web dashboard:
Global KPIs, heatmaps, statistics
Delay anomalies and risk alerts
Staff, roles, agencies, couriers
Tariffs & pricing
Payments & invoices
Notifications & templates
Support tickets & dispute resolution
AI analytics (ETA performance, route optimization results)

🔷 SECTION QR CODE – VERSION PRO POUR FIGMA AI
1. QR Code Generation (Agent UI)
When registering a parcel:
The system generates a unique Parcel ID (ex: CAM-2025-000123)
A QR code is automatically generated from this identifier
The UI must show:
QR preview
Print button
Label layout preview (small sticker + large label versions)
Option to re-generate label
Provide a “Print Label” modal with:
Label size (Small, Medium, Large)
Toggle: Include human-readable ID
Agency stamp placeholder
2. QR Code Scanning UI (Agents, Couriers, Hubs)
Create a full scanning interface for:
Counter agents (ACCEPT step)
Pickup agents (COLLECT step)
Hub sorting agents (ARRIVAL, DEPARTURE)
Delivery agents (OUT_FOR_DELIVERY, DELIVERED)
Agency release (PICKED_UP_AT_AGENCY)
Scanning interface must include:
Live camera preview
Flashlight toggle
“Scan successful” animation
Parcel summary card (status, sender, destination)
Next action button depending on context
Error state (Invalid QR, Not assigned to you, Already scanned)
3. QR Code Timeline Events
Each scan updates the parcel timeline.
UI must show:
Event type badge
Time & date
Agent identity
Location (agency/hub + map icon)
Optional proof (signature/photo/OTP)
Device used (mobile scanner, desktop camera)
Provide timeline components that the developer can reuse.
4. QR Code for Pickup Requests (Temporary QR)
For home pickup:
Before the agent arrives, the user receives a temporary QR code
It is used by the pickup agent to:
Identify the request
Confirm they are scanning the correct parcel
Link temporary QR → final parcel QR
UI must include:
Temporary QR display screen for clients
“Show QR to the pickup agent”
Countdown validity timer
Replace with final QR after validation
5. QR Code on Receipts and Invoices
Every payment and shipment confirmation generates a receipt containing:
Parcel ID
QR Code for quick verification
Link to public tracking
Download PDF button
Figma must generate:
Digital receipt layout
Printable receipt layout
