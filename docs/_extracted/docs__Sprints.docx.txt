SPRINT 1 — Authentication & Security
🎯 Sprint Goal
Implement secure user authentication using JWT and OTP (SMS), and create the foundation for all authorization.
📌 User Stories
US1: As a client, I want to register using my phone & password.
US2: As a user, I want to log in securely so I can access my dashboard.
US3: As the system, I want to send OTP codes for phone verification.
US4: As the system, I want to validate OTP codes.
US5: As the system, I want to issue and validate JWT tokens.
🛠 Tasks
Create AuthController
Create AuthService & AuthServiceImpl
Implement RegisterClientRequest, AuthResponse DTOs
Implement BCrypt password hashing
Implement OTP generation + store + expiration
Implement JWT generation & validation
Configure SecurityConfig + JWTFilter
Add error handling & validation
Unit testing
✔ Acceptance Criteria
Registration returns valid JWT
Login returns JWT if credentials are correct
OTP expires after configured time
Invalid token → 401
Password stored hashed, never in plain text
⚠ Risks / Impediments
Third-party SMS provider failures
JWT clock drift
Wrong spring security configuration
OTP collisions (must ensure randomness)
📦 Deliverables
Fully functional /api/auth/register, /login, /send-otp, /verify-otp
SecurityConfig
JWT authentication system
