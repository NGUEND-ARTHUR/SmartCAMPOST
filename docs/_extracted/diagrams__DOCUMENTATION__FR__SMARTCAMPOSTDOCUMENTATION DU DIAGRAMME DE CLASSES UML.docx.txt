SMARTCAMPOSTDOCUMENTATION DU DIAGRAMME DE CLASSES UML
1. Introduction
Le diagramme de classes UML est l’un des outils fondamentaux de la conception orientée objet.Il permet de représenter la structure statique du système SmartCAMPOST, en identifiant toutes les classes, leurs attributs, leurs méthodes (opérations) ainsi que les relations existant entre elles.
Ce diagramme constitue le lien entre le modèle logique de données (MLD) et la modélisation objet du système.Il assure la cohérence entre la base de données, la logique métier et les fonctionnalités prévues dans les cas d’utilisation.

2. Objectifs du diagramme de classes
Les principaux objectifs du diagramme de classes sont :
Identifier les entités métiers principales du système SmartCAMPOST ;
Définir les relations et dépendances entre ces entités ;
Déterminer les attributs et comportements (méthodes) de chaque classe ;
Préparer la phase de programmation orientée objet (Java, Dart, etc.) ;
Garantir la cohérence entre les données et les fonctionnalités décrites dans le système.

3. Aperçu général du système SmartCAMPOST
Le système SmartCAMPOST vise à moderniser la gestion des envois postaux de la CAMPOST grâce à des fonctionnalités intelligentes.Il permet aux clients de :
Créer et suivre leurs envois de colis,
Effectuer des paiements numériques,
Générer un code QR pour le suivi,
Recevoir des notifications (SMS ou push),
Demander une collecte à domicile.
Les agents et coursiers peuvent quant à eux :
Enregistrer les colis à l’agence,
Scanner les codes QR pour mettre à jour les statuts,
Livrer les colis,
Capturer les preuves de livraison,
Gérer les paiements et les factures.

Le diagramme de classes représente la structure technique de ces interactions et des entités du système.

6. Justification de conception
Le modèle respecte les principes de la conception orientée objet.
Chaque classe correspond à une entité réelle du domaine postal.
Les énumérations garantissent la cohérence des statuts et types.
Les compositions (Colis → Preuve, Colis → Scan) traduisent une dépendance forte du cycle de vie.
Les associations représentent les liens logiques (Client, Paiement, Notification).
Les méthodes encapsulent les règles métiers principales (création, paiement, livraison).

7. Conclusion
Le diagramme de classes du système SmartCAMPOST offre une vue claire et complète de la structure du système.Il modélise les entités principales, leurs attributs, leurs relations, et les règles de gestion essentielles.Ce modèle constitue la base de référence pour le développement des modules applicatifs et garantit la traçabilité entre les exigences fonctionnelles et la conception technique.
