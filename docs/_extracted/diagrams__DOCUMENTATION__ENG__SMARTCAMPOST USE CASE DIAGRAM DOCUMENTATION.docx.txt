SMARTCAMPOST USE CASE DIAGRAM DOCUMENTATION
1. Introduction
The Use Case Diagram of SmartCAMPOST describes the interaction between external actors (users and systems) and the SmartCAMPOST platform.It illustrates how the different types of users — clients, staff, administrators, couriers, and external partners — interact with the system to achieve their objectives.This model also defines the scope of the system, its core functionalities, and the main relationships among the actors and use cases.

2. Objectives of the Use Case Diagram
The main objectives of this diagram are to:
Identify who interacts with the system (actors).
Define what each actor does (use cases).
Represent relationships and dependencies between use cases.
Provide a functional overview of the system before detailing the technical design.

3. Actors Description
Actor
Description
Client
The main user of the system. A client can send or receive parcels, request pickups, make payments, track shipments, and receive notifications.
Staff
Represents all postal employees authorized to handle operations. It includes agents and couriers.
Agent
A staff member at a CAMPOST branch who registers parcels, scans QR codes, updates statuses, and assists customers.
Courier
A staff member responsible for collecting parcels at client addresses and performing home deliveries.
Administrator
Manages users, agencies, tariffs, zones, and system configurations (payment, SMS, reports).
Finance Officer
Manages financial transactions, verifies payments, reconciles balances, and validates invoices.
Compliance Officer
Supervises compliance with AML/CFT rules, monitors anomalies, and freezes suspicious accounts.
Payment Provider
External system (Orange Money, MTN MoMo, CAMPOST Money) used to process payments and send payment callbacks.
SMS/Push Gateway
External system responsible for sending OTPs and delivery notifications to users.
Maps/Geolocation Service
External API used to validate addresses and provide navigation/routing support.
AI/Analytics Service
External AI module that predicts delivery times, detects anomalies, and supports optimization.
NPSI / USSD Switch (#237)
National payment and service aggregator enabling USSD menu access for feature phones.

Actor Hierarchy
Staff
 ├── Agent
 └── Courier
Agents and couriers inherit the permissions of the generic staff role, but each has specific operational tasks.

4. List of Use Cases (with Descriptions)
Client Use Cases
Use Case
Description
Register Account
The client creates a new SmartCAMPOST account.
Verify Phone via OTP
Confirms user identity during registration. (included in Register Account)
Authenticate (Login)
Allows the client to log in with phone number and password.
Manage Profile
The client updates personal details or preferences (language, address, contact).
Create Parcel Request
Allows the client to register a parcel to send.
Describe Parcel
Enter parcel weight, dimensions, value, fragility. (included in Create Parcel Request)
Provide Sender/Recipient Address
Select or add addresses. (included in Create Parcel Request)
Quote Price
The system calculates cost based on tariff and zone. (included in Create Parcel Request)
Generate Tracking Reference
Generates a unique tracking number. (included in Create Parcel Request)
Generate QR Label
Optionally creates a QR code label for home printing. (extends Create Parcel Request)
Request Pickup at Home
The client asks for a home collection.
Validate Pickup Address (Geo)
Confirms pickup location using GPS. (included in Request Pickup)
Schedule Pickup Slot
Choose a time for collection. (included in Request Pickup)
Choose Delivery Option (Agency vs Home)
The client selects delivery mode.
Pay Home-Delivery Surcharge
Applies if the client switches to home delivery. (extends Choose Delivery Option)
Pay for Service
Client pays for parcel shipment.
Initiate Payment
Launches the transaction via MoMo, Orange, or CAMPOST Money. (included in Pay for Service)
Verify Payment Result
Confirms payment success/failure. (included in Pay for Service)
Issue Invoice
Generates the digital invoice. (included in Pay for Service)
Track Parcel
Client tracks parcel status in real time.
View Parcel Timeline (Scan Events)
Displays history of scan events. (included in Track Parcel)
Subscribe to Notifications
Client opts in for SMS/push alerts. (extends Track Parcel)
Download Invoice/Receipt
Accesses and downloads invoice as PDF.
Open Support Ticket
Submits a complaint or inquiry.

6. Conclusion
The Use Case Diagram provides a comprehensive view of how SmartCAMPOST functions from the user’s perspective.It defines every interaction between internal users, external systems, and the platform’s processes.This model ensures that all functional requirements — from parcel creation, QR tracking, payment, delivery, to AI-based analytics — are captured clearly before development.
